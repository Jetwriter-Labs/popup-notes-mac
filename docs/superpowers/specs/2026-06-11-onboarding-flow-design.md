# Popup Notes — First-Run Onboarding Flow Design

- **Date:** 2026-06-11
- **Status:** Approved (design phase) — ready for implementation planning
- **Supersedes:** the **launch-at-login consent `NSAlert`** introduced by
  `cc4294f` and the **shortcut tip crammed into that alert** (`cc12f9e`).
  Everything else from
  [`2026-06-09-multiple-notes-swiftdata-design.md`](2026-06-09-multiple-notes-swiftdata-design.md)
  still holds.
- **Still true:** native-first, zero third-party dependencies, menu-bar
  accessory app, non-activating floating panel, user-customizable global
  hotkey (default ⌃⌘N), sandboxed, App Review 2.4.5(iii) — launch-at-login
  is **never** enabled without explicit user consent.

## 1. Summary

Onboarding becomes **in-product**: no welcome window, no launch-moment modal.
On first launch the notes panel **auto-opens**, seeded with a **Welcome note**
that teaches the basics, and a thin **onboarding strip** at the bottom of the
panel carries the two one-time concerns — a **hotkey hint** that persists
until the user first triggers the panel with the hotkey, and an inline
**launch-at-login consent row** (Enable / Not Now). The product demos itself;
the strip rows retire themselves; the `NSAlert` is deleted.

Rationale (HIG ¶ alerts): modal alerts are for decisions that require stopping
the user. Neither "here's your shortcut" nor "start at login?" qualifies —
both are better as inline, self-dismissing UI.

## 2. Decisions locked this session

| Question | Decision | Consequence |
|---|---|---|
| Overall shape | **In-product onboarding** (Approach A) — no welcome window | Zero new window chrome; the popup demos the popup. |
| First launch | **Auto-show the panel** once (existing mouse-screen centering) | "Show, don't tell." Gated by a new `didRunFirstLaunchOnboarding` flag, so it also fires once for upgraders. |
| Teaching content | **Seeded Welcome note**, only when the DB is empty post-migration | First line *is* the title — demonstrates the title rule by existing. Legacy importers keep their content, get no seed. |
| Hotkey retention | **Hint row in a bottom onboarding strip**, shown until the user first fires the hotkey (or clicks its X) | The row's disappearance is the success feedback. Live-mirrors the current combo from `HotKeyStore`. |
| Hotkey failure | Hint row swaps to a **"shortcut unavailable — set a new one in Settings (⌘,)"** variant when registration fails | Replaces today's silent `NSLog`-only failure. |
| Launch-at-login consent | **Inline strip row** with Enable / Not Now buttons | Explicit click = 2.4.5(iii) consent. The `NSAlert` and `promptForConsentIfNeeded` are deleted; a one-time launch reconciliation disables any pre-consent-era silent default (legacy `didApplyFirstRunDefaults` installs). Reuses the `didPromptLaunchAtLogin` flag, so upgraders who answered are never re-asked. |
| Consent failure | If `SMAppService.register()` throws on Enable, the row still hides; failure is logged; Settings toggle shows truth | Accepted trade-off — no retry UI in v1. |
| Welcome note staleness | Seeded text embeds the combo at seed time and is **not** updated on later shortcut changes | It's the user's note once seeded; the strip is the live element. |

## 3. Goals / Non-goals

**Goals:** experience the popup within seconds of first launch; the hotkey
sticks (persistent-until-learned hint); in-editor basics taught by a seeded
note; consent without modals; upgraders inconvenienced at most once
(one auto-show, hint until first hotkey use).

**Non-goals (YAGNI now):** multi-step welcome window, "What's New" on
updates, re-showable onboarding/tour, analytics on onboarding completion
(the app has none, by promise), coach-mark overlays, retry UI for
`SMAppService` failures.

## 4. First-launch flow

```
applicationDidFinishLaunching
├── setActivationPolicy(.accessory)
├── migrateLegacyIfNeeded()                  (unchanged, runs first)
├── register hotkey → record `hotKeyRegistered`
└── runFirstLaunchOnboardingIfNeeded()
    ├── guard !didRunFirstLaunchOnboarding else return
    ├── set didRunFirstLaunchOnboarding = true
    ├── if DB is empty: seed Welcome note (NotesRepository.create + save)
    └── panel.show()                          (existing centering/show path)
```

No alert fires at launch — `LaunchAtLogin.promptForConsentIfNeeded` is
removed along with its call site.

### Welcome note content

Generated by `WelcomeNote.text(hotKeyDisplay:)` with the *current* combo's
display string (default ⌃⌘N at seed time):

```
Welcome to Popup Notes 👋

Press ⌃⌘N in any app to open this panel — Esc or a click outside dismisses it.

A few basics:
• The first line of a note is its title in the sidebar.
• ⌘N makes a new note · right-click a note to delete it.
• Notes save automatically and live only on this Mac.
• Settings (⌘,): change the shortcut, launch at login, export your notes.

Delete this note whenever you're done with it.
```

## 5. Onboarding strip

A thin full-width bar pinned to the bottom of the panel, below both
`NavigationSplitView` columns, rendered by a new `OnboardingStripView`.
It contains up to two independent, self-retiring rows (strip renders nothing
once both are retired):

1. **Hotkey hint** — visible while `!didUseHotkeyOnce && !didDismissHotkeyHint`.
   - Normal: *"Press ⟨combo⟩ in any app to open Popup Notes"* + an X button
     (accessibility label "Dismiss shortcut hint"; sets `didDismissHotkeyHint`).
   - `hotKeyRegistered == false`: *"Your shortcut is unavailable — set a new
     one in Settings (⌘,)"*.
   - The combo display live-updates when Settings rewrites the `hotKeyCombo`
     default (`@AppStorage` on the same defaults key `HotKeyStore` writes).
2. **Launch-at-login consent** — visible while `!didPromptLaunchAtLogin`.
   *"Start at login so your shortcut is ready after a restart"* with
   **Enable** (`LaunchAtLogin.isEnabled = true`) and **Not Now**; either
   sets `didPromptLaunchAtLogin = true`.

## 6. State (all `UserDefaults`)

| Key | New? | Set by | Read by |
|---|---|---|---|
| `didRunFirstLaunchOnboarding` | new | `AppDelegate` after auto-show | `AppDelegate` |
| `didUseHotkeyOnce` | new | hotkey closure in `AppDelegate` (guarded write) | strip |
| `didDismissHotkeyHint` | new | strip's X button | strip |
| `didPromptLaunchAtLogin` | reused | strip's Enable / Not Now | strip |
| `hotKeyRegistered` | new | `AppDelegate` after each register attempt (launch + `applyHotKey`) | strip |
| `hotKeyCombo` | existing | `HotKeyStore` | strip (display string) |

`defaults delete ai.jetwriter.popupnotes` resets the whole flow for testing.

## 7. Component changes

| Unit | Change |
|---|---|
| `OnboardingStripView` (new, `Views/`) | The strip; flags via `@AppStorage`; one small single-purpose file. |
| `WelcomeNote` (new, `PopupNotesCore`) | Pure `static func text(hotKeyDisplay: String) -> String`. Testable. |
| `AppDelegate` | Remove consent call; add `runFirstLaunchOnboardingIfNeeded()`; record `hotKeyRegistered`; set `didUseHotkeyOnce` in the hotkey closure. |
| `NotesView` | Embed the strip at the panel's bottom edge. |

## 8. Upgraders (existing installs)

- `didPromptLaunchAtLogin` already set → consent row never shows; never
  re-asked.
- `didRunFirstLaunchOnboarding` unset → **one** auto-show after the update;
  DB non-empty → no seeded note.
- Hint row shows until their first hotkey press — seconds for an active user,
  and X exists for the rest.

## 9. Edge cases & error handling

- **Legacy import** precedes the seed check → importers keep their content,
  no Welcome note.
- **Delete-all-notes later** → no re-seed (flag set); `NotesView`'s existing
  empty-DB auto-create covers the blank state.
- **Hotkey conflict at first launch** → failure variant in the hint row;
  recording a working combo in Settings flips `hotKeyRegistered` back.
- **Menu-bar-only users** keep the hint row until they use the hotkey or
  dismiss it — intended.
- **`SMAppService` throws on Enable** → logged, row hides, Settings toggle
  remains the source of truth (locked decision §2).
- **Seed save fails** → non-fatal; SwiftData autosave is the net.

## 10. Testing

**Automated — `PopupNotesCore` (Swift Testing, `./scripts/test-core.sh`):**
- `WelcomeNote.text` embeds the passed hotkey display string.
- Its first line is `"Welcome to Popup Notes 👋"` and `NoteTitle` derives it
  as the title.
- Repository seed: creating the welcome note in an in-memory container yields
  exactly one note with that first line (pattern: `NotesRepositoryTests`).

**Manual smoke (CLAUDE.md checklist gets updated):** reset defaults → launch →
panel auto-opens with the Welcome note and both strip rows → press the hotkey
→ hint row gone for good → click Enable → app appears in System Settings ▸
Login Items → relaunch → no auto-show, no strip → change the shortcut in
Settings to a taken combo → failure variant appears (after resetting the
hint flags) → export/import and Esc/click-outside behavior unchanged.
